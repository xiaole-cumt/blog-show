project_camera_to_image

从相机坐标系转为图像坐标系

其中深度变换是一种归一化操作，指的是将计算得到的向数坐标 / 深度值，保证符合如下图的透视投影特性。

project_camera_to_image_v2

在转换后的二维中可视化深度信息

对于二维图像来说，形如（H，W，C），第一个坐标轴为y，第二个为x。

PointCloud2Detph

点云图转换为深度图

pc: 点云数据，形状为(n, 3)，每一行代表一个点的x, y, z坐标。
K: 相机内参数矩阵，形状为(3, 3)，用于将三维点投影到二维图像平面。
orientation: 四元数表示的相机方向，用于确定相机的旋转状态。
position: 相机的位置，一般是一个三维向量。
gripper_length: 假设与机械臂夹爪相关，用于生成边界框。
init_pose: 初始姿态，可能是一个4x4的变换矩阵。
shape: 输出深度图像的形状，默认为512x512。

深度图到点云图

dmap: 深度图，表示每个像素的深度值。
K: (3x3)相机内参数矩阵，包含焦距和光心坐标。
orientation: (4,) 四元数表示相机的旋转，格式为(w, x, y, z)。
position: (3,) 相机在三维空间中的位置。
mask: 用于过滤点云，保留特定区域的点。
org_size: 原始深度图的尺寸，用于缩放调整。

  • x3: (x坐标 - 光心x) * 深度值 / 焦距x，然后缩放到原始宽度。
  • y3: (y坐标 - 光心y) * 深度值 / 焦距y，然后缩放到原始高度。
  • z3: 深度值，这里直接使用dmap。

抓取分析

dmap：深度图数据。
intrinsic：相机的内参数矩阵。
cameraori：相机的旋转矩阵，即相机的方向。
camerapos：相机的位置。

对于深度图

转换为点云再转换回深度图的流程充分利用了两种数据表示形式的优势：点云提供了三维空间的详细信息，便于进行空间变换和三维处理；深度图则保留了与原始视角一致的二维信息，便于后续的图像处理和分析。这种方法在机器人视觉和抓取任务中是一种常见的处理流程，旨在通过不同数据形式的互补优势来提高任务的准确性和可靠性。

接触面与点云匹配形式

contact_indexes记录当前接触点的索引，first_contact_pc_indexes是点云的索引

contacts: 包含接触点的集合，通常是一组坐标（x, y, z）。
scores: 每个接触点的评分或置信度，表示该点被认为是真实接触点的可能性。
pc: 点云数据，表示3D空间中的点。
contact_th: 接触阈值，用于决定接触点是否被视为有效的接触。
policy: 匹配策略，可选值为’soft’或’hard’，决定匹配的严格程度。

硬性匹配

基于欧几里得距离和距离阈值将接触点对与点云中的点进行匹配，选出最近邻

contacts: 接触点对的集合，形状为 [M x 3 x 2]，表示 M 对接触点，每个接触点有 3 个坐标（x, y, z）。
scores: 握持标签，形状为 [M x 1]，表示每个接触点对的评分或置信度。
pc: 点云数据，形状为 [N x 3]，表示点云中的 N 个点，每个点有 3 个坐标（x, y, z）。
contact_th: 距离阈值，用于判断接触点是否足够接近点云点。

软匹配

软匹配记录所有满足阈值条件的点，而硬匹配只记录满足阈值的最近点

balance_grasp_indexes

从给定的 scores数组中选择一定数量的正样本和负样本索引，以平衡正负样本的数量

SampleGraspData

init初始化根目录，getitem根据索引获取对应形状的抓取信息，readgras读取并处理特定形状的抓取标注数据

YCB76_Data

只用于测试集，固定相机视角
